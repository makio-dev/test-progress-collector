## 概要

Windowsのタスクスケジューラを使用して、テスト進捗集計ツール（EXE）を毎日自動的に実行し、最新のレポートを生成する手順を説明します。

### 定期実行のメリット

- 毎朝の朝会前に最新の集計レポートが自動生成される
- 手動実行の手間がなくなり、レポートの生成漏れを防止できる
- CLIモードで実行するため、GUIの操作が不要

---

## 前提条件

| 項目 | 要件 |
|------|------|
| OS | Windows 10 / 11 |
| EXEファイル | `aggregate_test_results.exe` がビルド済みであること |
| 入力データ | テストケースファイル・欠陥一覧ファイルが所定のフォルダに配置済みであること |
| ネットワーク | 入力ファイルがネットワークドライブにある場合、タスク実行時にアクセス可能であること |

## 事前準備

### 1. フォルダ構成の確認

定期実行用のフォルダ構成を以下のように整理します。

```
C:\TestProgressTool\                  ← ツールの配置先（任意の場所）
├── aggregate_test_results.exe        ← EXEファイル
├── run_daily.bat                     ← 定期実行用バッチファイル（この後作成）
├── input\                            ← テストケースファイル
│   ├── defects\                      ← 欠陥一覧ファイル
│   │   ├── 欠陥一覧_オンライン.xlsx
│   │   ├── 欠陥一覧_バッチ.xlsx
│   │   ├── 欠陥一覧_基盤.xlsx
│   │   └── 欠陥一覧_運用.xlsx
│   ├── オンライン\
│   │   └── ITB-*.xlsx
│   ├── バッチ\
│   │   └── ITB-*.xlsx
│   ├── 基盤\
│   │   └── ITB-*.xlsx
│   └── 運用\
│       └── ITB-*.xlsx
└── output\                           ← 出力先
    └── report.xlsx                   ← 生成されるレポート
```

> **ポイント**: EXEファイルと入力フォルダは同じドライブに配置することを推奨します。

### 2. 手動実行での動作確認

定期実行を設定する前に、コマンドプロンプトで手動実行して正常に動作することを確認してください。

```cmd
cd C:\TestProgressTool
aggregate_test_results.exe .\input -o .\output\report.xlsx ^
    --defect-online .\input\defects\欠陥一覧_オンライン.xlsx ^
    --defect-batch .\input\defects\欠陥一覧_バッチ.xlsx ^
    --defect-infra .\input\defects\欠陥一覧_基盤.xlsx ^
    --defect-ops .\input\defects\欠陥一覧_運用.xlsx
```

`output\report.xlsx` が正常に生成されることを確認してください。

## バッチファイルの作成

タスクスケジューラから直接EXEを呼び出すこともできますが、**バッチファイル経由で実行することを推奨**します。ログ出力やエラー処理を柔軟に設定できます。

### 基本版（run_daily.bat）

以下の内容でバッチファイルを作成してください。

```bat
@echo off
REM ============================================================
REM テスト進捗集計ツール 定期実行バッチ
REM ============================================================

REM 作業ディレクトリの設定
cd /d C:\TestProgressTool

REM 実行日時をログファイル名に使用
set LOGDATE=%date:~0,4%%date:~5,2%%date:~8,2%_%time:~0,2%%time:~3,2%%time:~6,2%
set LOGDATE=%LOGDATE: =0%

REM ログファイルの出力先
set LOGFILE=output\exec_log_%LOGDATE%.txt

echo ========================================>> %LOGFILE%
echo 実行開始: %date% %time%>> %LOGFILE%
echo ========================================>> %LOGFILE%

REM 集計ツールの実行
aggregate_test_results.exe .\input -o .\output\report.xlsx ^
    --week-from %date:~0,4%/%date:~5,2%/%date:~8,2% ^
    --week-to %date:~0,4%/%date:~5,2%/%date:~8,2% ^
    --defect-online .\input\defects\欠陥一覧_オンライン.xlsx ^
    --defect-batch .\input\defects\欠陥一覧_バッチ.xlsx ^
    --defect-infra .\input\defects\欠陥一覧_基盤.xlsx ^
    --defect-ops .\input\defects\欠陥一覧_運用.xlsx >> %LOGFILE% 2>&1

REM 実行結果の確認
if %ERRORLEVEL% EQU 0 (
    echo 実行成功: %date% %time%>> %LOGFILE%
) else (
    echo 実行失敗 (エラーコード: %ERRORLEVEL%): %date% %time%>> %LOGFILE%
)

echo ========================================>> %LOGFILE%
echo 実行完了: %date% %time%>> %LOGFILE%
echo ========================================>> %LOGFILE%
```

### カスタマイズのポイント

| 項目 | 変更箇所 | 説明 |
|------|----------|------|
| ツール配置先 | `cd /d C:\TestProgressTool` | EXEファイルの配置先に合わせて変更 |
| 週範囲 | `--week-from`, `--week-to` | 実行日当日を設定する例。固定期間にする場合は直接日付を指定 |
| 欠陥ファイル | `--defect-*` | 不要なチームの行は削除してください |
| 出力先 | `-o .\output\report.xlsx` | 出力ファイルのパスを変更可能（絶対パスも可） |

> **注意**: 週範囲の日付取得は `%date%` の形式がWindowsのロケール設定に依存します。上記は `YYYY/MM/DD` 形式の場合の例です。環境に合わせて調整してください。

## タスクスケジューラの設定

### 手順1: タスクスケジューラを開く

1. `Win + R` キーを押して「ファイル名を指定して実行」を開く
2. `taskschd.msc` と入力して「OK」をクリック

<!-- 画像: タスクスケジューラ_起動.png -->

### 手順2: 基本タスクの作成

1. 右側の「操作」パネルから「**基本タスクの作成**」をクリック

<!-- 画像: タスクスケジューラ_基本タスクの作成.png -->

2. 以下の情報を入力:

| 項目 | 入力値 |
|------|--------|
| 名前 | `テスト進捗集計_定期実行` |
| 説明 | `テスト進捗集計ツールを毎日自動実行し、レポートを生成します` |

3. 「次へ」をクリック

### 手順3: トリガーの設定

<!-- 画像: タスクスケジューラ_トリガー設定.png -->

1. 「**毎日**」を選択して「次へ」
2. 以下を設定:

| 項目 | 設定値 | 備考 |
|------|--------|------|
| 開始日 | 任意の日付 | 本日以降の日付 |
| 開始時刻 | `08:00:00` | 朝会前の時刻を推奨 |
| 間隔 | `1日` | 毎日実行 |

3. 「次へ」をクリック

> **ポイント**: 入力ファイルがネットワークドライブにある場合は、PCがネットワークに接続された後の時刻を設定してください。

### 手順4: 操作の設定

<!-- 画像: タスクスケジューラ_操作設定.png -->

1. 「**プログラムの開始**」を選択して「次へ」
2. 以下を設定:

| 項目 | 設定値 |
|------|--------|
| プログラム/スクリプト | `C:\TestProgressTool\run_daily.bat` |
| 引数の追加 | （空欄） |
| 開始 (オプション) | `C:\TestProgressTool` |

> **重要**: 「開始 (オプション)」には**バッチファイルがあるフォルダのパス**を指定してください。これにより、バッチファイル内の相対パスが正しく解決されます。

3. 「次へ」をクリック

### 手順5: 確認と完了

<!-- 画像: タスクスケジューラ_確認画面.png -->

1. 設定内容を確認
2. 「**[プロパティ] ダイアログを開く**」にチェックを入れる
3. 「完了」をクリック

### 手順6: 詳細プロパティの設定（推奨）

完了後に開くプロパティダイアログで、以下の設定を行います。

#### 「全般」タブ

| 項目 | 推奨設定 |
|------|----------|
| ユーザーがログオンしているかどうかにかかわらず実行する | チェックあり（※） |
| 最上位の特権で実行する | チェックなし |

※ ログオフ中も実行したい場合に設定します。パスワードの入力が求められます。

#### 「条件」タブ

<!-- 画像: タスクスケジューラ_条件タブ.png -->

| 項目 | 推奨設定 |
|------|----------|
| コンピューターをAC電源で使用している場合のみタスクを開始する | チェックなし |
| コンピューターがアイドル状態の場合のみタスクを開始する | チェックなし |

#### 「設定」タブ

| 項目 | 推奨設定 |
|------|----------|
| タスクが失敗した場合の再起動 | チェックあり |
| 再起動の間隔 | `5分` |
| 再試行の最大回数 | `3` |
| タスクを停止するまでの時間 | `1時間` |

設定が完了したら「OK」をクリックします。

<!-- 画像: タスクスケジューラ_タスク一覧_作成完了.png -->

## 動作確認

### 手動でタスクを実行して確認

1. タスクスケジューラの左側ツリーで「**タスクスケジューラ ライブラリ**」を選択
2. 作成した「テスト進捗集計_定期実行」タスクを右クリック
3. 「**実行する**」を選択

<!-- 画像: タスクスケジューラ_手動実行.png -->

### 確認項目

| 確認項目 | 確認方法 |
|----------|----------|
| タスクが正常終了したか | タスクスケジューラの「前回の実行結果」が `(0x0)` であること |
| レポートが生成されたか | `output\report.xlsx` のタイムスタンプが更新されていること |
| ログが出力されたか | `output\exec_log_*.txt` が生成されていること |
| レポートの内容が正しいか | Excelを開いてダッシュボード・欠陥サマリーの内容を確認 |

## ログの確認方法

### 実行ログ（バッチファイルのログ）

バッチファイルにより `output\exec_log_YYYYMMDD_HHMMSS.txt` が生成されます。

```
========================================
実行開始: 2026/03/10 08:00:01
========================================
  対象フォルダ: .\input
  ...
  ✅ 出力完了: .\output\report.xlsx
実行成功: 2026/03/10 08:00:15
========================================
実行完了: 2026/03/10 08:00:15
========================================
```

### エラーログ

ツール自体のエラーが発生した場合、`output\error_log_YYYYMMDD_HHMMSS.txt` が自動生成されます。

### タスクスケジューラの履歴

1. タスクスケジューラで対象タスクを選択
2. 下部の「**履歴**」タブを選択
3. 各実行の成功/失敗を確認

> **注意**: 履歴が表示されない場合は、右側の「操作」パネルで「**すべてのタスクの履歴を有効にする**」をクリックしてください。

## 設定の変更・削除

### 実行時刻の変更

1. タスクスケジューラでタスクをダブルクリック
2. 「トリガー」タブを選択
3. トリガーを選択して「編集」をクリック
4. 開始時刻を変更して「OK」

### 実行頻度の変更（例: 平日のみ）

1. 「トリガー」タブでトリガーを編集
2. 「毎日」から「**毎週**」に変更
3. 月曜〜金曜にチェックを入れる

### タスクの無効化

1. タスクを右クリック → 「**無効にする**」
2. 再開する場合は右クリック → 「**有効にする**」

### タスクの削除

1. タスクを右クリック → 「**削除**」
2. 確認ダイアログで「はい」をクリック

## トラブルシューティング

### Q: タスクが実行されない

**確認項目:**
- タスクが「有効」になっているか確認
- 「前回の実行結果」のエラーコードを確認
- PCの電源が入っていて、ログオンしている（または「ログオンしていなくても実行」を設定している）か確認

---

### Q: 前回の実行結果が `(0x1)` になる

**原因**: バッチファイルまたはEXEの実行でエラーが発生

**解決方法:**
1. `output\exec_log_*.txt` の内容を確認
2. バッチファイルのパスが正しいか確認
3. 「開始 (オプション)」が正しく設定されているか確認
4. コマンドプロンプトで手動実行して同じエラーが出るか確認

---

### Q: レポートが更新されていない

**原因**: 出力先のExcelファイルが開いたまま実行された

**解決方法:**
- 定期実行時刻にExcelファイルを開かないようにする
- または出力ファイル名に日付を含める設定にする（バッチファイルを修正）

```bat
REM 日付付きファイル名の例
set OUTDATE=%date:~0,4%%date:~5,2%%date:~8,2%
aggregate_test_results.exe .\input -o .\output\report_%OUTDATE%.xlsx ^
    --defect-online .\input\defects\欠陥一覧_オンライン.xlsx
```

---

### Q: ネットワークドライブの入力ファイルが読めない

**原因**: タスク実行時にネットワークドライブがマウントされていない

**解決方法:**
- バッチファイルの先頭でネットワークドライブを接続する:

```bat
net use Z: \\server\share /user:domain\username password
```

- または UNCパスを直接使用する:

```bat
aggregate_test_results.exe \\server\share\input -o .\output\report.xlsx ^
    --defect-online \\server\share\input\defects\欠陥一覧_オンライン.xlsx
```

---

### Q: 古いログファイルが溜まっていく

**解決方法:**
バッチファイルに古いログの削除処理を追加:

```bat
REM 30日より古いログファイルを削除
forfiles /p "C:\TestProgressTool\output" /m "exec_log_*.txt" /d -30 /c "cmd /c del @path" 2>nul
```

## 付録: コマンドラインでのタスク登録（上級者向け）

GUIを使わず、コマンドラインでタスクを登録することもできます。

```cmd
schtasks /create ^
    /tn "テスト進捗集計_定期実行" ^
    /tr "C:\TestProgressTool\run_daily.bat" ^
    /sc daily ^
    /st 08:00 ^
    /sd 2026/03/10 ^
    /rl HIGHEST ^
    /f
```

### オプション説明

| オプション | 説明 |
|-----------|------|
| `/tn` | タスク名 |
| `/tr` | 実行するプログラム/スクリプト |
| `/sc daily` | 毎日実行 |
| `/st 08:00` | 開始時刻 |
| `/sd` | 開始日 |
| `/rl HIGHEST` | 最上位の特権で実行 |
| `/f` | 既存タスクの上書き |
